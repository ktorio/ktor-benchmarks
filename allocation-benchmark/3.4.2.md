### Key Takeaways

**Improvements**

- **Header key lookups no longer allocate `CaseInsensitiveString` wrapper objects; `forEach` no longer allocates a `LinkedHashMap` iterator.**
  [KTOR-9383 / #5379](https://github.com/ktorio/ktor/pull/5379) ŌĆö `CaseInsensitiveMap` replaced with an open-addressing hash table that computes case-insensitive equality directly on `String` keys, and `StringValuesImpl` replaced with parallel arrays with direct iteration; saves ~1,500ŌĆō2,500 bytes per request across all engines, partially offset by slightly larger per-instance map object sizes (+100ŌĆō700 bytes in `Collections.kt` / `StringValues.kt`).

- **Routing trace interceptor skipped when TRACE-level logging is disabled, eliminating `RoutingResolveTrace` and related context allocations on the happy path.**
  [KTOR-9385 / #5381](https://github.com/ktorio/ktor/pull/5381) ŌĆö `RoutingRoot` now conditionally registers the trace interceptor only when TRACE logging is active; saves ~650ŌĆō1,000 bytes per request on all server engines.

- **Netty: redundant `cancel()` call no longer allocates `JobCancellationException` on the happy path.**
  [KTOR-9385 / #5381](https://github.com/ktorio/ktor/pull/5381) ŌĆö `NettyApplicationCall` checks `responseWriteJob.isCompleted` before calling `cancel()`; saves ~70 bytes per Netty request.
